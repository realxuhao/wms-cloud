## git代码提交规范  
```
<type>(<taskId>): <subject>
// 空一行
<body>
```  
* type为提交类别： 
  ```bash
  feat：新功能（feature）
  fix：修补bug
  docs：文档（documentation）
  style： 格式（不影响代码运行的变动）
  refactor：重构（即不是新增功能，也不是修改bug的代码变动）
  test：增加测试
  chore：构建过程或辅助工具的变动
  ```
* taskId：为关联到teambition任务id，必要。  
* subject：为本次提交代码简单描述，必要。  
  ![images/task-id.png](img.png)  
* body：为本次提交代码的详细描述，非必要，可多行
